# 认知层（#01-#10）——怎么看这件事

---

## #01 第一性原理
- **来源**：西方（亚里士多德 → 埃隆·马斯克）
- **核心问题**：如何从0理解事物
- **核心动作**：拆到不可再拆的基本事实，重新推导

**思考流程**：
1. 列出当前所有假设
2. 逐个追问"这一定是真的吗？"
3. 拆到不可再拆的基本事实
4. 从基本事实重新推导结论
5. 对比"重新推导的结论"和"原来的结论"

**关键提问**：
- 这个问题的本质是什么？
- 有哪些不能违背的基本真理？
- 如果从零开始，会如何设计？
- 当前方案是基于事实还是基于惯性？

**示例**：
```
问题：如何提高团队效率？
→ 当前假设："加班=更多产出"
→ 拆解：效率 = 有效产出 / 时间资源
→ 基本事实：注意力有限、沟通有成本、技能需要积累
→ 重新推导：不是"加班"，而是减少无效沟通、提高单点效率
→ 洞察：效率瓶颈不在时间，在注意力管理和流程设计
```

**组合接口**：#02 批判性思维、#04 奥卡姆剃刀、#21 庖丁解牛
**反面风险**：陷入无限拆解，无法落地

## #02 批判性思维
- **来源**：西方（苏格拉底 → 现代逻辑学）
- **核心问题**：这个结论靠得住吗
- **核心动作**：质疑假设，检验证据，找替代解释

**思考流程**：
1. 识别核心论点
2. 检查证据的充分性和可靠性
3. 揭露潜在假设和偏见
4. 评估逻辑推理的有效性
5. 考虑替代解释

**关键提问**：
- 这个结论基于什么证据？
- 证据可靠吗？充分吗？
- 有没有隐含假设？假设成立吗？
- 有没有其他解释？
- 我自己的偏见是什么？

**示例**：
```
观点："远程办公效率低下"
→ 证据：某公司报告称生产力下降20%
→ 质疑：样本大小？测量方法？是否有对照组？
→ 隐含假设：所有工作类型相同、管理方式不变
→ 其他解释：可能是管理方式需调整，而非远程本身的问题
→ 洞察：问题不在"远程"，而在"如何管理远程工作"
```

**组合接口**：#01 第一性原理、#05 认知偏差审计、#11 二阶思维
**反面风险**：变成杠精，否定一切

## #03 贝叶斯思维
- **来源**：西方（托马斯·贝叶斯 → 现代概率论）
- **核心问题**：我的判断该调整了吗
- **核心动作**：根据新信息持续更新判断概率

**思考流程**：
1. 确定当前的先验概率（初始信念）
2. 识别新信息/证据
3. 评估新信息对概率的影响
4. 更新后验概率（修正后的信念）
5. 设定"什么信息会让我再次改变判断"

**关键提问**：
- 在看到新信息之前，我的初始判断是什么？
- 这条新信息有多大的说服力？
- 我是否在过度调整（一个数据点就推翻全部）？
- 什么信息会让我继续更新判断？

**组合接口**：#02 批判性思维、#14 期望值思维、#48 时间换空间
**反面风险**：过度调整，失去立场

## #04 奥卡姆剃刀
- **来源**：西方（奥卡姆的威廉）
- **核心问题**：最可能的原因是什么
- **核心动作**：选最简单解释，如无必要勿增实体

**思考流程**：
1. 列出所有可能的解释
2. 按复杂度排序
3. 优先验证最简单的解释
4. 只有简单解释被证伪后，才考虑更复杂的

**关键提问**：
- 最简单的解释是什么？
- 这个复杂假设有额外证据吗？
- 我是否在过度解读？

**组合接口**：#01 第一性原理、#02 批判性思维、#10 抽象建模
**反面风险**：过度简化，忽略真实存在的复杂性

## #05 认知偏差审计
- **来源**：西方（丹尼尔·卡尼曼 → 行为经济学）
- **核心问题**：我看到的是事实还是偏见
- **核心动作**：系统检查自己是否被常见认知偏差影响

**常见偏差清单**：
- 确认偏差：只看支持自己观点的信息
- 沉没成本谬误：因为已经投入而不愿放弃
- 幸存者偏差：只看到成功案例
- 锚定效应：被第一个信息框住
- 可得性偏差：容易想到的≠概率高的
- 损失厌恶：失去的痛苦 > 获得的快乐

**关键提问**：
- 如果结论相反，我会怎么看这些证据？
- 如果没有之前的投入，我还会做同样选择吗？
- 我看到的是全貌还是被筛选过的样本？

**组合接口**：#02 批判性思维、#03 贝叶斯思维、#12 阴阳平衡
**反面风险**：自我怀疑过度，决策瘫痪

## #06 格物致知
- **来源**：华夏（《大学》朱熹解读）
- **核心问题**：这个东西到底是什么
- **核心动作**：深入事物内部，穷究其理

**思考流程**：
1. 选定一个具体事物/现象
2. 不断追问"为什么"，至少5层
3. 直到触及底层原理或本质规律
4. 将理解内化为自己的认知框架

**关键提问**：
- 它为什么是这样而不是那样？
- 它跟同类事物的本质区别是什么？
- 如果要用一句话教给外行，我会怎么说？

**示例**：
```
格物：为什么手机电池越来越不耐用？
→ 第1层：电池老化（化学降解）
→ 第2层：为什么化学会降解？（充放电循环导致电极材料损耗）
→ 第3层：为什么不能设计不损耗的？（热力学第二定律，熵增不可逆）
→ 致知：所有储能设备都有寿命，关键不是"让它不老"，而是"让它老得慢"
```

**组合接口**：#07 见山是山、#08 名实之辨、#46 厚积薄发
**反面风险**：钻牛角尖，脱离实际

## #07 见山是山（三阶段）
- **来源**：华夏（青原惟信禅师）
- **核心问题**：我的理解到了哪一层
- **核心动作**：识别认知所处的阶段

**三阶段**：
1. **见山是山**：初始状态，看什么就是什么（朴素认知）
2. **见山不是山**：学习后，发现事物复杂/矛盾/多面（解构期）
3. **见山还是山**：穿透复杂后，看到本质但不再简化（重构期）

**关键提问**：
- 我现在的理解，是"还没学"还是"学完了又看透了"？
- 我是在用复杂装深刻，还是真的理解了简单背后的复杂？
- 这个"简单"的结论，是经过复杂推导后的简单，还是懒得想的简单？

**组合接口**：#06 格物致知、#09 类比思维、#24 知行合一
**反面风险**：故弄玄虚，自我感动

## #08 名实之辨
- **来源**：华夏（名家/公孙龙 → 逻辑分析传统）
- **核心问题**：这个名字背后的真实是什么
- **核心动作**：区分"叫什么"和"是什么"，撕标签看本质

**思考流程**：
1. 识别当前使用的标签/概念
2. 追问"这个标签到底指什么？"
3. 找到标签背后的实体/现象/规律
4. 判断标签是准确描述还是误导

**关键提问**：
- 当我们说"XX"的时候，我们实际在说什么？
- 这个标签/概念/名头背后，实际发生了什么？
- 如果换一个名字，我的判断会不同吗？

**组合接口**：#06 格物致知、#22 法术势、#34 情理法
**反面风险**：过度解构，否定一切表达

## #09 类比思维
- **来源**：西方（亚里士多德 → 认知科学）
- **核心问题**：别的地方有答案吗
- **核心动作**：跨领域结构映射

**思考流程**：
1. 识别问题的核心结构（不是表面特征）
2. 寻找具有相似结构的其他领域
3. 映射关系和原理
4. 应用到原问题

**常用类比库**：
- 生物学：进化、生态系统、免疫、共生
- 物理学：杠杆、惯性、共振、熵
- 经济学：供需、边际效应、机会成本
- 军事：战略、后勤、士气、情报

**关键提问**：
- 这个问题的结构像什么？
- 在哪个领域，类似问题已经被解决了？
- 类比的边界在哪里？哪里映射不过去？

**组合接口**：#07 见山是山、#10 抽象建模、#41 顺势而为
**反面风险**：类比不当，似是而非

## #10 抽象建模
- **来源**：西方（数学/科学方法论）
- **核心问题**：这件事的底层模型是什么
- **核心动作**：从具体案例提取通用结构

**思考流程**：
1. 收集多个具体案例
2. 找到共同的结构/模式
3. 用简洁的形式表达这个模式
4. 验证：用模型能否解释新案例？

**关键提问**：
- 这些案例背后的共同模式是什么？
- 如果只能用一个公式/框架表达，会是什么？
- 这个模型在什么条件下会失效？

**组合接口**：#01 第一性原理、#04 奥卡姆剃刀、#09 类比思维
**反面风险**：过度抽象，脱离现实
