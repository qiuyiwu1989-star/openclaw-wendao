## #61 系统边界界定

**用途：** 任何复杂问题的第一步——先把"看什么"想清楚。

**三问：**
1. 这个问题涉及哪些要素？（列出来）
2. 边界在哪里？哪些算"系统内"、哪些算"系统外"？
3. 这个系统在更大的系统里是什么位置？

**陷阱提醒：**
- 边界画太大 → 分析瘫痪
- 边界画太小 → 看不到关键影响
- 默认做法：先画窄，再根据需要扩展

## #62 反馈循环识别

**用途：** 区分"直线因果"和"循环因果"——真实世界几乎都是循环的。

**核心判断：**
- **正反馈（放大）**：A变多 → B变多 → A更多。趋势自我强化。
- **负反馈（稳定）**：A变多 → B变多 → A变少。系统自我调节。

**识别方法：**
- 如果你发现"越做越有效"或"越做越差" → 有正反馈
- 如果你发现"做了又回到原样" → 有负反馈在拉回
- 如果你发现"效果延迟很久才出现" → 中间有延迟

**教练式提问：**
- "这件事的'结果'会不会反过来影响'原因'？"
- "你做了A之后，有没有什么力量在把A的效果拉回来？"

## #63 根因 vs 症状分离

**用途：** 解决"问题反复出现"的根本方法。

**区分标准：**
- **症状**：你看到的表面问题（员工流失、客户投诉、收入下降）
- **根因**：维持这个症状存在的系统结构

**判断方法：**
- 如果治了之后"好了又犯" → 你治的是症状
- 如果治了之后"永远不再出现" → 你治的是根因
- 根因通常不在问题出现的地方（"原因和结果在时间空间上经常不在一起"）

**教练式提问：**
- "这个问题之前出现过吗？当时怎么解决的？后来为什么又来了？"
- "如果这个问题消失了，是什么让它不再回来？"

## #64 意外后果预判

**用途：** 在做决策之前，预判"好心办坏事"的可能性。

**预判流程：**
1. 我的行动会直接影响哪些要素？
2. 这些要素的变化，会通过连接影响哪些其他要素？
3. 影响链条里有没有反馈循环？会被放大还是被稳定？
4. 有没有延迟？（延迟导致"当时觉得没问题，后来才出事"）

**教练式提问：**
- "如果你做了这个决定，谁/什么会受影响？"
- "那些受影响的，会不会反过来影响你？"
- "有没有什么后果是现在看不出来，但三个月后会显现的？"

## #65 杠杆点识别

**用途：** 找到"小改变带来大效果"的关键位置。

**七个常见杠杆点（从弱到强）：**
1. **参数调整**（最弱）：改数字、调预算——效果有限
2. **缓冲区大小**：库存、现金流储备——影响稳定性
3. **存量-流量结构**：改变积累方式——慢但深刻
4. **延迟长度**：缩短反馈延迟——让系统更快响应
5. **负反馈强度**：增强自我调节能力——让系统更稳定
6. **正反馈强度**：增强或削弱放大效应——改变增长/崩溃方向
7. **系统规则**（最强）：改变激励机制、权力结构、信息流——重新设计游戏

**教练式提问：**
- "在这个系统里，如果只能改一件事，改什么影响最大？"
- "你一直在推的那个地方，是不是系统里最不重要的变量？"
- "有没有什么地方你一直没动过，但其实它是关键？"

## #66 系统基模匹配

**用途：** 把眼前的复杂问题匹配到经典模式，快速定位分析方向。

**常用基模速查：**

| 基模 | 一句话 | 识别信号 |
|------|--------|---------|
| 增长极限 | 一开始好推，后来推不动 | "推不动了" |
| 转移负担 | 吃止痛药不吃治疗药 | "停药就复发" |
| 目标侵蚀 | 降低标准来达标 | "差不多就行" |
| 公地悲剧 | 人人薅、没人管 | "谁都不管" |
| 富者愈富 | 赢家通吃 | "头部吃肉" |
| 饮鸩止渴 | 短期解、长期毒 | "当时觉得解决了" |
| 恶性竞争 | 螺旋升级 | "军备竞赛" |
| 投资不足 | 不舍得投→长不动 | "产能跟不上" |

**教练式提问：**
- "你描述的这个过程，像不像'越吃止痛药越离不开'？"
- "是不是像跑步机——跑得很快但一直在原地？"

## #67 干预设计与副作用评估

**用途：** 找到杠杆点之后，设计具体行动并预判副作用。

**设计框架：**
1. **干预对象**：在哪个杠杆点下手？
2. **干预方式**：增强/削弱哪个循环？
3. **预期效果**：什么时间、什么程度能看到变化？
4. **可能副作用**：这个干预会触发什么其他循环？
5. **监控指标**：用什么来判断干预是否生效？
6. **退出机制**：如果副作用太大，怎么退回来？

**教练式提问：**
- "你准备推的这个力，会不会在别的地方产生反作用力？"
- "如果效果不如预期，你打算什么时候收手？"
- "有没有一个'最短观察期'——在这个时间内不判断成败？"

## #68 六顶思考帽（平行思考编排）

**用途：** 把"对抗性思考"变成"平行思考"——同一时间大家往一个方向想，不吵架，不掐创意。

**核心原则：** 不是六个人辩论，是一个人/一个团队的六种思维模式，分开使用。

**六顶帽子速查：**

| 帽子 | 干什么 | 教练式提问 |
|------|--------|-----------|
| 🔵 蓝帽 | 管理过程、定议程、总结 | "我们现在要讨论什么？目标是什么？" |
| ⚪ 白帽 | 只看事实和数据 | "撇开观点，我们实际知道什么？缺什么信息？" |
| 🔴 红帽 | 说感受和直觉，不解释 | "你的第一反应是什么？不需要理由，直说。" |
| 🟡 黄帽 | 找好处和价值 | "这件事好的一面是什么？为什么可能行？" |
| ⬛ 黑帽 | 找风险和问题 | "哪里可能出错？为什么可能行不通？" |
| 🟢 绿帽 | 提创意，不批评 | "如果没有任何限制，还有什么做法？" |

**场景化帽子顺序：**

| 场景 | 推荐顺序 | 理由 |
|------|---------|------|
| 头脑风暴 | 蓝→白→绿→红→黄→黑→蓝 | 先发散（绿）再收敛（黑） |
| 决策评估 | 蓝→白→黄→黑→绿→红→蓝 | 先看价值再看风险 |
| 问题解决 | 蓝→白→黑→绿→红→蓝 | 先诊断（黑）再开药（绿） |
| 团队冲突 | 蓝→红→白→黄→黑→绿→蓝 | 先放情绪（红）再讨论 |
| 快速评估 | 蓝→黄→黑→绿→蓝 | 省略白帽和红帽，30分钟搞定 |

**教练式提问：**
- "如果从最乐观的角度看，这件事最大的价值是什么？"
- "现在不准批评——如果没有任何约束，你会怎么改？"
- "你现在的反对意见，是基于事实（白帽）还是担心（黑帽）？还是直觉不喜欢（红帽）？"

**个人使用技巧：** 一个人思考时，可以按顺序"切换人格"。每换一顶帽子时，物理上站起来换个位置或者拿个实物提示，强制大脑切换模式。
