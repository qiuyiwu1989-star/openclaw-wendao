# 行动层（#21-#30）——怎么做

---

## #21 庖丁解牛
- **来源**：华夏（《庄子·养生主》）
- **核心问题**：从哪里下手
- **核心动作**：顺着结构纹理拆解，不硬砍

**思考流程**：
1. 不急着动手，先观察整体结构
2. 找到事物的"纹理"——自然的分解线
3. 顺着纹理拆解，每一步都自然顺畅
4. 拆到最小可执行单元

**关键提问**：
- 这件事的自然结构是什么？
- 顺着哪条线切下去最省力？
- 拆到什么粒度就可以动手了？

**示例**：
```
问题：重构一个50万行的代码库
→ 硬砍：重写（风险极大）
→ 庖丁：先画模块依赖图 → 找到最独立的模块 → 从边缘往核心逐步重构
→ 洞察：不是"一口气改完"，是"顺着依赖关系，找到最安全的切入点"
```

**组合接口**：#01 第一性原理、#25 最小行动单元、#30 破局点思维
**反面风险**：拆太碎，失去全局

## #22 法术势
- **来源**：华夏（韩非子）
- **核心问题**：哪个点投入最小产出最大
- **核心动作**：找杠杆，四两拨千斤

**三个维度**：
- **法**：规则/制度——改规则能改变行为
- **术**：技巧/方法——用对方法事半功倍
- **势**：趋势/位置——借势比造势容易

**关键提问**：
- 改什么规则能自动解决这个问题？
- 有没有一个技巧能让效率翻倍？
- 现在的趋势是什么？我能借什么势？

**组合接口**：#21 庖丁解牛、#27 杠杆放大、#30 破局点思维
**反面风险**：杠杆找错，满盘皆输

## #23 无为而治
- **来源**：华夏（老子）
- **核心问题**：哪些事不该我做
- **核心动作**：审计无效干预，该放手就放手

**思考流程**：
1. 列出当前所有干预/管理动作
2. 逐个评估：这个干预真的有效吗？还是在制造问题？
3. 找到"做了反而更差"的干预
4. 停止无效干预，让系统自行运转

**关键提问**：
- 如果我什么都不做，会怎样？
- 这个干预是在解决问题还是在制造新问题？
- 有没有"系统的自愈能力"被我压制了？

**组合接口**：#24 知行合一、#29 节奏控制、#59 断舍离
**反面风险**：放任失控，丧失主动

## #24 知行合一
- **来源**：华夏（王阳明）
- **核心问题**：知道的做到了吗
- **核心动作**：扫描认知和行动之间的差距

**思考流程**：
1. 列出你"知道"但没"做到"的事
2. 分析差距原因：是不想做？不能做？还是不知道怎么做？
3. 找到"知行差距"最大的那一条
4. 设计最小行动来弥合差距

**关键提问**：
- 我真正相信的是什么？我的行为反映了这个信念吗？
- 哪些"知道"其实只是"听说过"？
- 如果真的相信，为什么没做？

**组合接口**：#07 见山是山、#23 无为而治、#25 最小行动单元
**反面风险**：变成自我批评，焦虑

## #25 最小行动单元
- **来源**：西方（精益创业/MVP理念）
- **核心问题**：第一步具体做什么
- **核心动作**：拆到最小可执行步骤，立刻能做

**思考流程**：
1. 定义最终目标
2. 倒推：达到目标需要什么前置条件？
3. 继续倒推，直到找到"今天就能做"的最小步骤
4. 这个步骤必须：5分钟内能开始、24小时内能完成

**关键提问**：
- 如果只能做一件事，做哪件？
- 这一步完成后，下一步自然显现吗？
- 能不能今天就做完第一步？

**组合接口**：#21 庖丁解牛、#26 快速试错、#30 破局点思维
**反面风险**：陷入细节，忽略全局

## #26 快速试错
- **来源**：西方（硅谷精益方法论）
- **核心问题**：能先试试吗
- **核心动作**：小步验证，快速得到反馈

**思考流程**：
1. 把假设列出来
2. 设计最小实验来验证每个假设
3. 设定"成功/失败"的判断标准
4. 快速执行，看结果，迭代

**关键提问**：
- 最快的验证方式是什么？
- 什么结果算"这个方向是对的"？
- 试错成本能承受吗？

**组合接口**：#25 最小行动单元、#28 路径依赖、#29 节奏控制
**反面风险**：试错太频繁，没有积累

## #27 杠杆放大
- **来源**：西方（阿基米德 → 投资/管理学）
- **核心问题**：什么是最高ROI的事
- **核心动作**：找到能用小资源撬大结果的点

**思考流程**：
1. 列出所有可能的行动
2. 评估每个行动的：投入量 × 产出量 × 成功率
3. 找到"投入产出比"最高的
4. 集中资源在杠杆点上

**关键提问**：
- 如果只有一小时/一千块，我会花在哪？
- 哪个行动能同时解决多个问题？
- 有没有"做了之后其他事自动变简单"的事？

**组合接口**：#22 法术势、#30 破局点思维、#45 长期主义
**反面风险**：追求捷径，忽视基础

---

## #28 路径依赖
- **来源**：西方（制度经济学/布莱恩·阿瑟）
- **核心问题**：过去的选择如何限制了现在
- **核心动作**：考虑历史路径对当前选择的约束

**思考流程**：
1. 梳理事情发展到今天的关键节点
2. 识别每个历史选择带来的约束
3. 区分"可以改变的约束"和"锁定的约束"
4. 对锁定约束：接受它；对可变约束：挑战它

**关键提问**：
- 如果没有历史包袱，我会怎么选？
- 哪些约束是"真实的"，哪些是"惯性想象的"？
- 有没有"路径重置"的机会？

**组合接口**：#26 快速试错、#44 后发制人、#47 节点判断
**反面风险**：过度受限于过去，不敢突破

---

## #29 节奏控制
- **来源**：华夏（军事战略/《孙子兵法》）
- **核心问题**：什么时候快什么时候慢
- **核心动作**：分配快慢节奏

**思考流程**：
1. 识别事情的不同阶段
2. 判断哪些阶段该快（抢占、执行）
3. 判断哪些阶段该慢（观察、积累）
4. 设计"快-慢-快"的节奏方案

**关键提问**：
- 现在是该加速还是该慢下来？
- 有没有"太快反而坏事"的阶段？
- 我的节奏是自然的还是被焦虑驱动的？

**组合接口**：#23 无为而治、#42 天时判断、#49 复利思维
**反面风险**：节奏判断错误

---

## #30 破局点思维
- **来源**：西方（系统论/格拉德威尔）
- **核心问题**：哪个点突破了全局就活了
- **核心动作**：找量变到质变的关键点

**思考流程**：
1. 分析系统的当前状态
2. 找到"差一点就突破"的临界点
3. 评估：需要多少额外投入才能推过临界点？
4. 集中资源突破这一点

**关键提问**：
- 现在离突破还差什么？
- 哪个变量的变化会产生不成比例的效果？
- 有没有"99度的水只差1度就沸腾"的地方？

**组合接口**：#21 庖丁解牛、#22 法术势、#27 杠杆放大
**反面风险**：找错破局点，白费力气
