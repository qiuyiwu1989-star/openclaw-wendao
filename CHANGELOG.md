# 问道 — 版本记录

- **v4.8.0** (2026-07-21) — **护栏 + 通话可视 + 音色可选**：① 公开接口限流（chat 30/min、tts·asr 80/min per IP，超限友好 429），保护成本；② 通话屏显示最近一轮对话（你说的 + 问道的回答），不再"盲对话"，也能看出 ASR 有没有听错；③ 顶栏齿轮选音色（苏打/白桦 男，冰糖/茉莉 女），本地记忆
- **v4.7.0** (2026-07-21) — **安全边界 + 收尾分寸**（让教练"真的好"第一层）：危机识别——用户表达严重痛苦时停止一切教练动作、以有温度的方式导向全国心理援助热线 12356，凌驾于"短/不给答案"之上；会收尾——想清楚了就落地、放人走，不无限追问；接得住"你直接说"——真诚表态而非机械挡回；单场连贯。均为提示词层（SKILL v3.6 + persona 顶层安全条），已实测三种行为
- **v4.6.0** (2026-07-21) — **语音识别换 MiMo ASR（修国内不可用）**：浏览器 Web Speech API 走谷歌服务器、国内被墙，通话永远卡在"在听你说"。改为 MediaRecorder+VAD 录音 → MiMo `mimo-v2.5-asr` 转写。附：麦克风权限被拒的健壮处理（不无限重试 + 提示）、进通话预热 prompt cache（把冷启从 ~4s 压到 ~2s）
- **v4.5.0** (2026-07-20) — **通话模式**：顶栏电话键进入免手连续语音对话，说完自动接话、说完自动接着听；罗盘 orb 三态呼吸（在听/在想/在说），点 orb 打断。只在"听"阶段开麦避免回采自身声音（无回声）。已上线 https://wendao.zaowuyun.com
- **v4.4.0** (2026-07-20) — **语音提速**：定位真延迟瓶颈是巨型系统提示词（~88k 字冷缓存重算）。语音模式三件套——关思考（`thinking:disabled`）+ 精简提示词（砍掉 50k 字元件细节）+ 句级流式 TTS（整句一出就读）。语音首句热缓存 ~1.5s。打字/深度模式保留完整提示词+思考
- **v4.3.0** (2026-07-20) — **流式语音 + 男声**：TTS 改为流式（pcm16 边收边放，首字节即开声，Web Audio 无缝拼接），默认音色换成男声「苏打」。保留整段 wav 作降级（Web Audio 不可用时自动回落）
- **v4.2.0** (2026-07-19) — **对话引导引擎**：确立「逼×化」方法论宪法（西方拆算防统 / 华夏人时行），落地 11 招引导动作库 + 六信号挖掘引擎（追着"卡"走）+ 螺旋下沉流程。问道从"套 5 模块模板"升级为"看人下菜、一层一停"的对话引擎。新增 METHODOLOGY.md 设计锚，同步写入 SKILL.md 对话层
- **v4.1.0** (2026-07-19) — **语音对话**：回答自动朗读（小米 MiMo TTS，默认音色「冰糖」），可静音/重听；提示词改造为**短而准的口语教练**，一次推进一步、把问题踢回给用户，不再甩长篇五模块。后端切换到 MiMo 网关（对话走 /anthropic，语音走 TTS 模型）
- **v4.0.0** (2026-07-19) — 从「被调度的技能」升级为**独立的对话思维教练 Web App**（深脑下游应用）。Next.js 流式聊天界面，服务端把 SKILL.md/methods 加载为系统提示词（单一事实源）。技能大脑保持不变，仍可被其他 Agent 复用
- **v3.4.0** (2026-05-01) — 轻量模式：日常对话中的 1 句话点拨 + 反问。三种参与模式（轻量/标准/深度）
- **v3.1.0** (2026-04-05) — 正式命名「问道」。教练式交互流程：输入识别 + 5模块输出 + 隐藏方法引擎 + 高级模式
- **v3.0.0** (2026-04-05) — 60个结构化思维元件 + 方法调用引擎，中西融合工程化
- **v2.0.0** (2026-04-05) — 8种西方方法 + 24种华夏框架，中西融合
- **v1.0.0** (2025-03-20) — 初始版本，8种思考方法论
